## このフォークについて

本リポジトリは、アルゴリズムの標準的教科書である
CLRS（*Introduction to Algorithms*／邦題：**アルゴリズムイントロダクション**）
の章末問題解答をまとめた、以下のオープンソースリポジトリを
学習目的でフォークしたものです。

- [CLRS Solutions](https://github.com/walkccc/CLRS) （元のリポジトリ）

本フォークには、CLRS の問題に対する個人的な解答および考察メモを追加しています。
中学3年時に原著を独学で読み進めながら取り組んだもので、
アルゴリズムやデータ構造については理解を深めることを目的として、
既存実装に依存せず Python により自力で実装しました。

章末問題についても、多くは既存の解答を参照せずに試行錯誤しながら解いたものです。
その過程で、従来解答が提示されていなかった問題に対する解法を考えた例や、
既存解答の誤りに気づき修正を提案した例があり、
一部は Pull Request として上流リポジトリに提出しました。
解答の正確さだけでなく、考えた過程や別解の検討を重視して記録していました。

#### [章末問題13-2 赤黒木における join 操作](https://github.com/walkccc/CLRS/pull/158)

本件では、アルゴリズムイントロダクションに記載されている
**赤黒木における join 操作**について、実装および説明の整理・改善を行いました。

* **(c) の検討**
  部分木 $T_y$ を $T_y \cup {x} \cup T_2$ に置き換える処理について、
  親子ポインタを個別に書き換える方法ではなく、
  標準的な **TRANSPLANT 操作**を用いることで、
  **二分探索木の性質を保ったまま $O(1)$ 時間で置換できる**ことを明確にしました。
  これにより、実装の簡潔性と正当性が向上します。

* **(d) の検討**
  新たに挿入する節点 $x$ の色は **赤** とすることで、
  赤黒木の性質（1・3・5）を維持できることを示しました。
  その上で、残る性質（2・4）については
  標準の **INSERT-FIXUP** 手続きを適用することで
  **$O(\log n)$ 時間で回復できる**ことを整理しました。

本提案では、

* 構造変更（定数時間）
* 色調整・再平衡（対数時間）

を明確に分離し、赤黒木の不変条件を保つ実装方針を簡潔に示しています。

#### [演習問題23.2-5 Prim のアルゴリズムの計算量改善に関する考察](https://github.com/walkccc/CLRS/pull/199)

アルゴリズムイントロダクションにおける  
Prim のアルゴリズムの実行時間に関する問題について、  
辺重みが整数である場合の **データ構造選択による計算量改善**を整理し、
解答として提案しました。

具体的には、以下の2つのケースを区別して考察しています。

- **辺重みが 1 から |V| の整数の場合**  
  優先度付きキューとして **van Emde Boas 木**を用いることで、  
  Prim のアルゴリズムを **\(O(E \log \log V)\)** 時間で実行できることを示しました。  
  フィボナッチヒープによる実装と比較し、  
  疎グラフにおいてのみ漸近的改善が得られる点や、  
  多項式的改善ではないものの実装上のオーバーヘッドが低くなる可能性についても言及しています。

- **辺重みが 1 から定数 \(W\) の範囲に制限される場合**  
  各重みに対応する **二重連結リスト（バケット）**を用いることで、  
  優先度付きキュー操作を定数時間で行え、  
  Prim のアルゴリズム全体を **\(O(E)\)** 時間で実行できることを整理しました。

この Pull Request では、  
単に解答を述べるだけでなく、  
**入力制約に応じたデータ構造選択がアルゴリズム全体の計算量に与える影響**を
明確にすることを重視しています。
